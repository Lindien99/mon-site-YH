Workflow Branches

main ──●──────────────────●── stable \ / feature ●──●──●──●──/ dev
1. Créergit checkout -b ma-feature
2. CoderFaire les modifications
3. Commitgit add . && git commit
4. Poussergit push origin ma-feature
5. PROuvrir une Pull Request GitHub
6. MergerFusionner dans main après review
🔀

Pull Requests

Draft PREn cours, pas prête à être relue
ReviewUn relecteur doit approuver
Merge commitHistorique complet préservé
Squash mergeTous les commits → 1 seul
Rebase mergeHistorique linéaire propre
CloseAbandon sans fusion
Tip : lier une PR à une Issue avec Closes #n dans la description

GitHub Actions — CI/CD

# .github/workflows/deploy.yml name: Déploiement on: push: branches: [main] jobs: build: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - run: npm run build
WorkflowFichier YAML d'automatisation
JobGroupe d'étapes sur une machine
StepUne action ou commande
RunnerMachine virtuelle d'exécution
